## рҹ“– Overview

**DS NexusFlow** is a full-stack, production-ready analytics platform built for DS Group's last-mile delivery operations. It replaces manual Excel-based payout processing with a live, formula-accurate engine that ingests custom workbooks, computes hub-specific volumetric rider payouts, and surfaces executive-grade operational insights вҖ” all in a single, beautifully designed web interface.

> The platform mirrors DS Group's proprietary payout formula model (v2 workbook), including SKU-level volumetric calculations, fulfillment ratios, binary cancellation compensation logic, and hub-threshold gating.

---

## вңЁ Features

### рҹ“Ҡ Executive Dashboard
Real-time KPI cards displaying total payout, active riders, fleet fulfillment rate, cancellation compensation, and delivered volume. Includes payout trend charts, order status distributions, and a top-rider leaderboard.

### рҹ“Ұ SKU Breakdown Engine
Deep SKU-level analytics per order вҖ” unit volumes, delivery status classification (`Fully Delivered` / `Partially Delivered` / `Not Delivered`), bulky SKU flagging, and volume contribution percentages.

### рҹ’° Order Payout Calculator
Mirrors the Excel formula engine exactly:
- **Fulfillment ratio** = `totalDeliveredVolume / totalOrderVolume`
- **Binary cancellation multiplier** based on hub GPS drift threshold
- **Final payout** = `basePayout Г— fulfillmentRatio` (or `basePayout Г— cancellationMultiplier` for cancelled orders)

### рҹ§ҫ Rider Summary Module
Per-rider aggregates: daily / weekly / monthly payouts, genuine delivery attempts, failed attempts, cancellation compensation, and average fulfillment ratios вҖ” all computed from raw order data.

### вҡ пёҸ Cancellations Intelligence
Separates `Cancelled With Valid Attempt` from `Cancelled Without Valid Attempt` using per-hub GPS drift thresholds. Configurable compensation logic per hub.

### рҹ“Ҙ Excel Ingestion (SheetJS)
Upload custom `.xlsx` workbooks. The parser service auto-detects DS Group sheet schemas, enriches SKU data from an internal registry, and hydrates the global Zustand store вҖ” no manual data entry required.

### вңҸпёҸ Manual Order Entry
Add, edit, and remove individual orders through a form-based UI. Fully reactive вҖ” all downstream KPIs recalculate instantly via Zustand selectors.

### вҡҷпёҸ Settings & Hub Configuration
Admin interface to configure hub-level GPS drift thresholds, base payout rates, bulky SKU volume thresholds, and reference dates. Changes propagate globally and trigger a full engine recalculation.

### рҹ”” Notification System
Ephemeral, session-based notification bell with read/unread badge tracking. Fires contextual alerts for manual additions, bulk uploads, and compensation triggers.

### рҹ”Қ Global Search & Filter
Cross-page search that filters orders and riders in real-time, with a result-count banner on the dashboard.

```bash

# 3. Start the development server
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

## рҹ§® Payout Formula Reference

NexusFlow precisely replicates the DS Group v2 payout model:

```
SKU Level
в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
OrderedSkuVol        = unitVolume Г— orderedQty
DeliveredSkuVol      = unitVolume Г— deliveredQty
VolumeContribPct     = deliveredSkuVol / orderDeliveredVol
SkuPayoutContrib     = basePayout Г— volumeContribPct

Order Level
в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
FulfillmentRatio     = totalDeliveredVol / totalOrderVol
CancellationMult     = driftKm вүӨ hubThreshold ? 1 : 0
FinalOrderPayout     = basePayout Г— fulfillmentRatio       (delivered/partial)
                     = basePayout Г— cancellationMult       (cancelled)

Fleet Level
в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
FleetFulfillmentRate = ОЈ deliveredVol / ОЈ orderedVol       (non-cancelled)
CancellationRate     = cancelledOrders / totalOrders
```
